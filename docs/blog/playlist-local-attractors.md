# 旧播放模型为什么会落进局域吸引子

一条播放轨迹可以看成一个很小的动力系统。

当前歌曲给出一个位置。候选歌曲构成附近的状态空间。模型把当前位置和每个候选点之间的关系转成权重，再用一次随机抽样选出下一步。播放结束后，新的歌曲变成新的当前位置，系统重新计算下一步。这个规则很短，却足以产生稳定的轨迹形状。

旧模型的问题正出在这里。它把“这首歌像不像当前歌”当成了主要转移规则，却没有把“这条轨迹已经在同一个区域停留太久”建成状态。于是它有局部几何，没有路径记忆；有随机抽样，没有反复播放后的恢复力；有候选数量，没有跨吸引盆迁移的机制。

日志里能看到这个结构。候选数可以有一百多个，抽样也确实在运行，概率也没有退化成确定选择。但轨迹仍然会围着少数歌曲来回走。某些歌曲的相似度接近 1，局部排名接近 0，权重自然集中在它们身上。当前歌曲一旦进入这个区域，下一首又很容易留在同一区域。下一轮的 anchor 仍然在这个区域，权重又被重新推向同一批邻居。系统没有死锁，却形成了一个小的吸引盆。

这种现象不会被“候选很多”自动解决。状态空间大，只说明系统有很多可能位置；转移规则如果总把概率质量压回局部邻域，实际可达的轨迹仍然很窄。随机数也不能保证探索。随机抽样只是在当前分布里抽样，分布本身已经被局部相似度塑形时，随机性会在吸引盆内部产生抖动，而不是把系统带到另一条轨迹。

单向量表示会放大这个问题。音乐里有多个概念成分：音色、节奏密度、情绪、空间感、人声位置、动态范围、制作年代、旋律运动。旧模型把这些东西压成一个向量，再用一个距离解释下一步。压缩后的邻近关系可能是真实的，但它只给出一种合成后的近邻。一个歌曲在“情绪”上接近，在“节奏”上也接近，就会成为强邻居；如果当前轨迹连续进入这类强邻居，系统会把它解释成自然延续，而不是重复。

这就是旧模型最核心的错误：它学到了局部相似性，却没有学到轨迹应该怎样移动。

一个播放系统需要区分两个职能。模型负责描述歌曲之间的感知关系。播放会话负责管理轨迹的生命史。相似度模型不应该知道“用户刚刚听过哪些歌”，也不应该为了跳出循环而篡改嵌入空间。会话层也不应该重新解释音乐概念。它只需要维护当前会话历史，把已经播放过且没有被 liked 标记的歌曲暂时移出候选集。

这个会话历史相当于给状态空间加入一个简单的不可立即返回区域。歌曲被播放后，如果它只是普通候选，就暂时不能再次成为下一步。轨迹因此被迫离开刚刚访问过的小区域。liked 歌曲例外，因为 liked 是用户明确留下的可重复吸引点；它不能被普通排除规则吞掉。liked 也不应该变成最高权重或最低权重，它应该保留在中间概率带里，让用户偏好可见，同时不把系统拉成单点循环。

这个规则看起来很小，但它改变的是动力系统，而不是局部权重公式。旧模型每一步只看当前 anchor 和候选云。新规则让会话层携带一段路径证据，再把这段证据作用在候选资格上。模型仍然计算音乐相似性；会话仍然只管理播放历史；组合层负责把两者放在同一次选择里。边界清楚，系统才不会把“防重复”这种轨迹语义塞进 embedding、cache、fallback 或日志里。

还有一个边缘情况决定了这个规则能不能闭合：如果所有候选都已经在会话历史里，排除规则必须退回原候选集。否则系统会把“防重复”变成终态，导致没有下一首可播。这个退回只说明候选空间已经被历史约束耗尽，继续播放优先于制造空队列。这个退回的作用域只在候选资格层，不改变歌曲相似度，也不改变 liked 的含义。

从这个事故里得到的经验很直接。

第一，推荐分布和播放轨迹属于两个问题。分布描述当前位置附近的选择倾向，轨迹描述系统随时间移动的形状。只修分布，不能保证轨迹不陷入吸引盆。

第二，相似性和新鲜度需要分属不同职能。相似性越稳定，越需要一个独立职能保存会话访问证据。没有这段证据，模型会把重复误认为连贯。

第三，随机性本身不构成探索策略。抽样只能服从当前权重。如果权重由局部邻域主导，随机抽样会生成局部变化，无法保证全局迁移。

第四，liked 是用户留下的特殊吸引点。它可以抵抗短期排除，但不能变成无限重复的许可。它的概率应该可采样、可出现、不过分支配。

第五，多概念音乐表示不应该急着合成一个最终向量。不同概念维度可以保留各自的向量，再让播放策略决定如何组合距离。这样系统可以沿情绪连续、沿节奏跳转，也可以沿音色保持、沿动态范围释放。轨迹移动会变成多个概念分量的组合，而不是单一距离的滚动。

第六，trace 只能解释现象，不能参与现象。日志里的 probability、similarity、best_similarity、local_rank_fraction 很有用，因为它们让局域吸引子可见。但删除 trace 后，播放结果必须不变。trace 一旦影响选择，就变成了隐藏行为来源。

旧模型的错误可以压缩成一句话：它把歌曲空间里的近邻关系当成了播放时间里的运动规律。

播放推荐需要两层规则。第一层回答“哪些歌在感知上接近”。第二层回答“系统已经怎样走过，现在还应该怎样走”。局域吸引子来自这两层混在一起后的自然结果。修复不需要把模型弄得更神秘；只要把路径历史交还给会话，把音乐关系交还给模型，把两者的组合交还给播放选择，轨迹就会重新获得离开局部区域的能力。
